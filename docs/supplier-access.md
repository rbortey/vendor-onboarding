# Supplier Access Guidance

Supplier access is approved per engagement and should be limited to the systems required for the work order.

## Access Classes

| Class | Typical use | Approval |
|---|---|---|
| Portal only | CRM, invoice, or ticket follow-up | Operations owner |
| Remote support | Scheduled support window for a named system | IT support lead |
| File exchange | Document handoff, reconciliations, exports | Department head |

## Notes

- Remote support access is time boxed and should be reviewed after each support window.
- Vendor access packs are issued by IT support after the access request is approved.
- Supplier users should not receive broad internal routes by default.
- If a supplier needs access to more than one host, document the reason in the request.

## Current Review Items

- Finance reconciliation suppliers sometimes need short maintenance windows during month-end close.
- Access should be limited to the relevant host or segment wherever possible.
- Operations should keep the ticket reference and supplier contact in the access request.
